**CLAUDE.md**

Project: 
CLI + MCP for OLX (Polish market), generated by Printing Press.

Goals: 
- Fetch job listings from OLX (mainly production / warehouse / logistics categories). 
- Store data in a local database (SQLite). 
- Analytics: detect companies or accounts that post many job listings in these categories. 
- Prioritize extraction of: company name, company address, job title, phone number, email address, short job description. 
- Provide a simple CLI interface that is easy to integrate later with external systems (Pipedrive, REGON, OpenClaw, etc.).

Directory structure: 
- `~/projects/printing-press/olx/src` – source code only (Go, MCP, config). 
- `~/projects/printing-press/olx/bin` – binaries only (`olx-pp-cli`, `olx-pp-mcp`, etc.). 
- `~/projects/printing-press/olx/data` – runtime data: SQLite, cache, CSV/JSON exports. 
- `~/projects/har/olx` – HAR files with real OLX HTTP requests (input for Printing Press).

Rules: 
- Use Go (idiomatic, readable, no unnecessary abstraction layers). 
- Do not create new directory trees like `~/printing-press/library` unless explicitly requested – stick to the structure above. 
- Default path for database and working files should be `./data` relative to the OLX project root. 
- Operate only on publicly available OLX data, read‑only (no modifications on the OLX side). 
- Do not touch files or directories outside `~/projects/printing-press/olx` and `~/projects/har/olx` unless the user explicitly asks for it.
