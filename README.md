# OLX Sales-Intelligence CLI

This repository contains the `olx-pp-cli`, a reverse-engineered CLI and MCP server for the OLX.pl job-listings API.

## Features

- **`sync`** — pull public OLX.pl job listings (production / warehouse / logistics by default) into a local SQLite store.
- **`companies` / `jobs`** — read-only analytics to surface employers posting many listings.
- **`enrich`** — enrich detected companies with Polish registry data (NIP, KRS, REGON, address, legal form) via the [bizraport.pl](https://bizraport.pl) API. Credentials go in `[bizraport]` of the config or `BIZRAPORT_EMAIL`/`BIZRAPORT_PASSWORD`; the API bills per returned row, so responses are cached per KRS and runs are gated by `--limit` / `--budget-pln`.
- **`export`** — dump jobs/companies (incl. registry columns) to CSV or JSON for downstream tooling (Pipedrive, REGON, …).
- **MCP server** — the same workflow as agent tools; `--read-only` / `READ_ONLY_MODE` withholds the live-network tools (`olx_sync`, `olx_enrich`).

Stored ids are namespaced (`olx:<id>`) with a `source` column, so the local DB can later hold records from more than one source.

## Directory Structure

- `src/` - Contains the Go source code for the CLI and MCP server, generated by [CLI Printing Press](https://github.com/mvanhorn/cli-printing-press) and customized manually. Check out [`src/README.md`](src/README.md) for more details.
- `docs/` - Contains project documentation, research notes, and build logs (e.g., [`docs/build-log.md`](docs/build-log.md)).
- `data/` - Directory for local data storage (like SQLite databases) and data exports.

## Building the Project

To build the CLI and MCP server binaries, run the following commands:

```bash
cd src
make build-all
```

The compiled binaries will be available in the `src/bin/` directory.

## Running Tests

To run the tests for the project:

```bash
cd src
make test
```
